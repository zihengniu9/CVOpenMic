# 安全策略

## 报告安全问题

请不要通过公开 Issue 披露 API Key 泄露、任意代码执行、隐私数据暴露或可绕过的资源限制。

在仓库启用 GitHub Private Vulnerability Reporting 后，请优先通过仓库的 **Security → Report a vulnerability** 私密报告。启用前，可联系仓库维护者并仅提供问题概述，不要在首次联系中附带真实个人数据或密钥。

## 数据边界

- 上传文件在内存中解析，应用不会主动将简历写入项目目录。
- 邮箱、手机号和身份证号会在发送给模型前自动脱敏。
- 脱敏并不等于完全匿名化；工作经历和教育背景仍可能具有识别性。
- 模型供应商如何处理请求数据，以其服务条款和隐私政策为准。

## 密钥处理

- `.env`、`.env.*` 和 `.streamlit/secrets.toml` 已被 Git 与 Docker 忽略。
- 不要在 Issue、日志、截图或构建参数中公开密钥。
- 如果密钥可能已进入 Git 历史、镜像层或共享文件，请立即在供应商控制台轮换。
