# Default Word resume layout

Use this as the default visual architecture for editable DOCX delivery. It is inspired by compact, high-information Chinese recruiting resumes: plain, credible, ATS-readable, and easy to edit.

## Visual system

- Use a black-and-white, single-column layout with no full-width color banner, oversized metric cards, sidebars, decorative icons, or background blocks.
- Center the candidate name. Place target role and contact details in one or two compact centered lines below it.
- Omit age, gender, and photo by default. Add a photo only when the user explicitly requests it or the target market clearly expects one.
- Use one restrained Chinese sans-serif font family throughout. Keep body text approximately 8.5–10 pt and section headings approximately 10.5–12 pt.
- Use narrow but printable A4 margins, normally 12–16 mm. Keep sufficient line spacing for readability; density must not become crowding.

## Information architecture

Use this default early-career order:

`Identity header → Education → Work experience → Project experience → Research/publications → Awards → Skills`

Do not replace formal education or work history with a metric dashboard or summary cards. If a short summary is necessary for a career pivot, keep it to two lines and do not let it displace core sections.

## Section and entry structure

- Format each module as a bold heading followed by a thin horizontal rule.
- Education row: `Institution | Degree/field | Dates`, optionally followed by one compact evidence line.
- Work or project header: `Organization/project | Team or scope | Role` with the date aligned to the right.
- Use real Word bullets below each header. Lead each bullet with a bold conclusion, deliverable, or supported metric, followed by action and context.
- Keep bullets to one or two lines when possible. Use nested bullets only for genuinely complex, modular ownership; do not create decorative nesting.
- Keep dates, organization names, titles, punctuation, and alignment consistent across all entries.

## Density and pagination

- Let Word paginate naturally. Do not hard-break between education, work, projects, or research.
- Prefer one page when legible. Use two pages when one page would require tiny type, omitted entries, or cramped line spacing.
- Rebalance entries across pages before shrinking text below the readable range.
- A section heading must stay with at least one complete entry. An entry header must stay with at least its first bullet.
- Treat a blank lower half as a failed layout. Fix it in the DOCX by removing hard breaks, moving complete entries, or adjusting spacing.

## Editable artifact rules

- Keep all text as native Word paragraphs, runs, real lists, or genuinely tabular rows.
- Do not flatten the page into an image or use positioned text boxes for primary content.
- Render the DOCX to PDF for preview; never create an independent PDF layout.
- After every content or formatting revision, regenerate the PDF and inspect every page.
